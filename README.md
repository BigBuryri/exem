## Модули 1–3 — Банкетам.Нет

Я сделала веб-портал для бронирования банкетных помещений: зал, ресторан, летняя и закрытая веранда.

**Стек:** React, Vite, Tailwind, Node.js, Express, PostgreSQL, Sequelize, JWT.

**Что реализовала:**
- регистрация и вход пользователей;
- подача заявок на бронирование с валидацией;
- личный кабинет и список своих заявок;
- админ-панель (смена статусов, сортировка, поиск);
- слайдер на главной, блок контактов, блок качества;
- ER-диаграмма и SQL-схема базы данных.

### Запуск

```bash
npm run install:all
npm run dev
```

- Сайт: http://localhost:3000  
- API: http://localhost:5000  
- Админ: **Admin26** / **Demo20**

### База данных

![ER-диаграмма](Модули%201-3%20(основная%20часть)/database/диаграмма.jpg)

**Пояснение к ER-диаграмме**

Я спроектировала базу из двух связанных таблиц:

| Таблица | Назначение |
|--------|------------|
| **users** | Пользователи системы (логин, пароль, ФИО, телефон, email, роль `user` / `admin`) |
| **applications** | Заявки на бронирование банкета |

**Связь:** один пользователь (`users`) может иметь много заявок (`applications`) — связь **1 : N** (один ко многим).  
Внешний ключ `applications.user_id` ссылается на `users.id`, при удалении пользователя заявки удаляются каскадно (`ON DELETE CASCADE`).

**Поля заявки:**
- `status` — статус: новая, банкет назначен, завершена;
- `type` — тип заявки (бронирование банкета);
- `data` — данные формы в формате JSON (тип помещения, дата, способ оплаты и т.д.);
- `admin_comment`, `rating`, `review` — комментарий администратора и отзыв пользователя после завершения.

Файлы схемы: `Модули 1-3 (основная часть)/database/init.sql`, `er-diagram.drawio`.

## Модуль 4 — Код безопасности

Я сверстала прототип мобильного приложения **430×932 px**: 2 экрана, HTML + CSS, 4-колоночная сетка.  
Тексты и структура — по материалам задания (`media/Текст.pdf`).

**Что сделала:**
- экран «Главная»: описание компании, блоки статистики, поиск, кнопки «Войти» и «Зарегистрироваться»;
- экран «Продукты»: каталог решений по категориям с кнопкой «Подробнее» у каждого продукта;
- единый стиль, минималистичная вёрстка, удобные зоны нажатия;
- кликабельные переходы между экранами через нижнее меню **Главная ↔ Продукты**.

**Файлы:** `Модуль 4 (вариативная часть)/index.html`, `products.html`, `css/style.css`.

Открыть локально: `Модуль 4 (вариативная часть)/index.html`
